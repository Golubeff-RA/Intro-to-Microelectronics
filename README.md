# Лабораторная 1
<p>Лаба_1 - консольное приложение на .Net. Для операций над матрицами используется пакет MathNet (если что его надо скачать будет с помощью NuGet). </p> 
<p>При запуске принимает в качестве аргумента командной строки путь до файла схемы.
Если у вас возникли проблемы с запуской (FileNotFound exception), то возможно в вашем пути слэши не в ту сторону смотрят и чтобы это не переписывать можно запустить программу просто перенеся файл.json на бинарь, тогда Windows укажет его в качестве первого агрумента. </p>
<p>Работа с программой. После обсчёта схемы и вычисления матриц для системы ДУ вы будете должны ввести начальный вектор, время конца симуляции и шаг симуляции (все в формате 1,234567, т. е. дробные числа в русской нотации через запятую)</p>

- Формат файла. В файле задаются 5 массивов элементов (резисторы, конденсаторы, индуктивности, источники напряжения, источники тока), массив перемнных состояния (X), массив выходов (Y). 
- Все элементы имеют "unique_id" от 0 до n-1 (число элементов) без пробелов в номерах, иначе крах, "input_node", "output_node" (char) - входная и выходная нода, по которым соединяются элементы.
- Описание элемента X и Y схожее: указывается "unique_id" элемента на котором смотрим напряжение или ток "state_type": "current" или "voltage", "annitation" - подпись для графика.
```json
{
    "resistors": [
        {
            "unique_id": 0,
            "resistance": -0.2,
            "input_node": "B",
            "output_node": "C"
        }
    ],

    "capacitors": [
        {
            "unique_id": 1,
            "capacity": 2,
            "input_node": "C",
            "output_node": "A"
        }
    ],

    "inductors": [
        {
            "unique_id": 2,
            "inductivity": 2,
            "input_node": "A",
            "output_node": "B"
        }
    ],

    "state_valiables": [
        {
            "annotation" : "U_C",
            "unique_id": 1,
            "state_type": "voltage"
        },
        {
            "annotation" : "I_L",
            "unique_id": 2,
            "state_type": "current"
        }
    ],

    "outputs": [
        {
            "annotation" : "I_R",
            "unique_id": 0,
            "state_type": "current"
        },
        {
            "annotation" : "I_C",
            "unique_id": 1,
            "state_type": "current"
        }
    ]
}
```
